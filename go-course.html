Introducción a Go

Historia, instalación y tu primer programa

Go (también llamado Golang) es un lenguaje de programación compilado, tipado estáticamente, creado en Google en 2007 por Robert Griesemer, Rob Pike y Ken Thompson. Fue diseñado para resolver los problemas de escala que Google enfrentaba con sus sistemas: compilaciones lentas, dependencias complicadas y dificultad de mantenimiento.

⚡

Compilado y rápido

Compila directamente a binario nativo, sin VM. Performance cercana a C/C++.

🔒

Tipado estático

Los tipos se verifican en compilación, eliminando errores en runtime.

🧵

Concurrencia nativa

Goroutines y channels son primitivas del lenguaje, no librerías.

📦

Simplicidad

Solo 25 keywords. La curva de aprendizaje es una de las más bajas.

¿Por qué Go para microservicios?

Go es la opción preferida para microservicios por varias razones clave:

  • Binary único: el compilado es un ejecutable standalone, perfecto para Docker
  • Arranque ultrarrápido: milisegundos vs segundos en JVM
  • Bajo consumo de memoria: ideal para múltiples instancias
  • Librería estándar poderosa: servidor HTTP, JSON, criptografía incluidos
  • Adoptado por: Docker, Kubernetes, Terraform, GitHub, Uber, Twitch

Instalación

  1. Descarga Go desde go.dev/dl para tu sistema operativo
  2. Instala el paquete. Go se instala en /usr/local/go (Linux/macOS) o C:\Go (Windows)
  3. Agrega al PATH: export PATH=$PATH:/usr/local/go/bin
  4. Verifica: go version → debería mostrar go version go1.22.x

Variables y Tipos

Sistema de tipos, declaraciones y constantes

Declaración de Variables

Go ofrece tres formas de declarar variables, cada una con su uso ideal:

variables.go 
package main

import "fmt"

func main() {
    // Forma 1: declaración explícita con var
    var nombre string = "Elkin"
    var edad   int    = 30

    // Forma 2: inferencia de tipo con var
    var activo = true

    // Forma 3: declaración corta (short declaration) — solo dentro de funciones
    ciudad := "Bogotá"
    pi     := 3.14159

    // Declaración múltiple
    var (
        host string = "localhost"
        port int    = 8080
    )

    fmt.Println(nombre, edad, activo, ciudad, pi, host, port)
}
🚀
Best practice: Usa := dentro de funciones (más idiomático). Reserva var para variables a nivel de paquete o cuando necesitas el zero value explícito.

Zero Values

En Go, toda variable declarada sin inicializar recibe un zero value automático. No hay valores indefinidos ni null pointer exceptions por variables no inicializadas.

Tipo Zero Value Ejemplo
int, int8, int64... 0 var x int // x == 0
float32, float64 0.0 var f float64 // f == 0.0
string "" var s string // s == ""
bool false var b bool // b == false
pointer, slice, map, chan nil var p *int // p == nil

Strings

strings.go 
package main

import (
    "fmt"
    "strings"
    "strconv"
)

func main() {
    s := "Hola, Go"

    // Operaciones básicas
    fmt.Println(len(s))              // 8 (bytes, no chars)
    fmt.Println(s[0])                // 72 (byte 'H')
    fmt.Println(string(s[0]))        // "H"
    fmt.Println(s[0:4])             // "Hola"

    // String multilínea (raw string literal)
    json := `{
        "nombre": "Elkin",
        "ciudad": "Bogotá"
    }`
    fmt.Println(json)

    // Funciones del paquete strings
    fmt.Println(strings.ToUpper(s))        // "HOLA, GO"
    fmt.Println(strings.Contains(s, "Go")) // true
    fmt.Println(strings.Split(s, ", "))    // ["Hola" "Go"]
    fmt.Println(strings.TrimSpace("  hi "))

    // fmt.Sprintf — interpolación de strings
    nombre := "Elkin"
    msg := fmt.Sprintf("Bienvenido, %s! Versión %d", nombre, 2)
    fmt.Println(msg)

    // Conversión número ↔ string
    num := strconv.Itoa(42)          // int → string
    n, _ := strconv.Atoi("123")      // string → int
    fmt.Println(num, n)
}

Runes y UTF-8

En Go, los strings son secuencias inmutables de bytes en UTF-8. Un rune (alias de int32) representa un punto de código Unicode. La distinción bytes/runes es clave para manejar correctamente texto con emojis, tildes o caracteres de idiomas asiáticos.

runes.go 
package main

import (
    "fmt"
    "unicode/utf8"
)

func main() {
    s := "Hola, 世界 🐹"

    // len() devuelve BYTES, no caracteres
    fmt.Println(len(s))                    // 17 (bytes)
    fmt.Println(utf8.RuneCountInString(s)) // 10 (caracteres reales)

    // range sobre un string devuelve (índice en bytes, rune)
    for i, r := range s {
        fmt.Printf("byte[%2d] = %c  (U+%04X)\n", i, r, r)
    }
    // byte[ 0] = H  (U+0048)
    // byte[ 6] = 世 (U+4E16)  ← 世 ocupa 3 bytes, el índice salta
    // byte[ 9] = 界 (U+754C)

    // Convertir a []rune para acceso por posición de carácter
    runes := []rune(s)
    fmt.Println(len(runes))       // 10
    fmt.Println(string(runes[7])) // "🐹"

    // Literal rune — comillas simples
    var a rune = 'A'    // 65
    var n rune = 'ñ'    // 241
    var e rune = '🐹'   // 129017
    fmt.Println(a, n, e)

    // Invertir string correctamente (con []rune, no bytes)
    r2 := []rune("Héroe")
    for i, j := 0, len(r2)-1; i < j; i, j = i+1, j-1 {
        r2[i], r2[j] = r2[j], r2[i]
    }
    fmt.Println(string(r2)) // "oéreH" ✅ tildes intactas
    // Si invirtiéramos bytes: resultado corrupto con tildes
}
💡
Regla práctica: Usa len(s) para tamaño en bytes (rendimiento), utf8.RuneCountInString(s) para contar caracteres. Convierte a []rune cuando necesites acceder o manipular caracteres individuales con tildes o emojis.

Control de Flujo

if, for, switch y defer

Go tiene una particularidad importante: solo existe una estructura de bucle: for. No hay while ni do-while, pero for los reemplaza a todos.

if / else

control.go 
// if estándar — sin paréntesis en la condición
edad := 25
if edad >= 18 {
    fmt.Println("Mayor de edad")
} else if edad >= 13 {
    fmt.Println("Adolescente")
} else {
    fmt.Println("Niño")
}

// if con inicialización (muy común en Go)
// La variable solo existe dentro del bloque
if x := calcular(); x > 0 {
    fmt.Println("Positivo:", x)
} else {
    fmt.Println("No positivo:", x)
}

// Patrón típico para manejo de errores
if err := hacerAlgo(); err != nil {
    log.Fatal(err)
}

for — El único bucle

bucles.go 
// Forma 1: clásico (como en C/Java)
for i := 0; i < 5; i++ {
    fmt.Println(i)
}

// Forma 2: como while
n := 1
for n < 100 {
    n *= 2
}

// Forma 3: infinito (equivalente a while true)
for {
    if condicion { break }
}

// Forma 4: range — iterar colecciones
frutas := []string{"mango", "guanábana", "lulo"}
for i, fruta := range frutas {
    fmt.Printf("%d: %s\n", i, fruta)
}

// Si no necesitas el índice, usa _
for _, fruta := range frutas {
    fmt.Println(fruta)
}

// range sobre un mapa
colores := map[string]string{"rojo": "#FF0000", "azul": "#0000FF"}
for clave, valor := range colores {
    fmt.Printf("%s = %s\n", clave, valor)
}

// break y continue funcionan igual que en otros lenguajes
// También existe "labeled break" para salir de bucles anidados

switch

switch.go 
// switch NO requiere break — solo ejecuta el caso que coincide
dia := "lunes"
switch dia {
case "lunes", "martes", "miércoles", "jueves", "viernes":
    fmt.Println("Día laboral")
case "sábado", "domingo":
    fmt.Println("Fin de semana")
default:
    fmt.Println("¿?")
}

// switch sin expresión (como if/else escalable)
hora := 14
switch {
case hora < 12:
    fmt.Println("Buenos días")
case hora < 18:
    fmt.Println("Buenas tardes")
default:
    fmt.Println("Buenas noches")
}

// Type switch — muy útil con interfaces
func describir(i interface{}) {
    switch v := i.(type) {
    case int:
        fmt.Printf("Entero: %d\n", v)
    case string:
        fmt.Printf("String: %s\n", v)
    case bool:
        fmt.Printf("Bool: %v\n", v)
    default:
        fmt.Printf("Tipo desconocido: %T\n", v)
    }
}

defer

defer pospone la ejecución de una función hasta que la función que la contiene retorne. Los defers se ejecutan en orden LIFO (último en entrar, primero en salir). Es la herramienta de Go para garantizar limpieza de recursos.

defer.go 
package main

import (
    "fmt"
    "os"
)

func leerArchivo(nombre string) error {
    f, err := os.Open(nombre)
    if err != nil {
        return err
    }
    defer f.Close() // ← se ejecuta cuando leerArchivo() retorne, SIEMPRE

    // ... procesar archivo con seguridad
    return nil
}

// Orden LIFO de defers
func lifo() {
    defer fmt.Println("tercero")
    defer fmt.Println("segundo")
    defer fmt.Println("primero")
    fmt.Println("ejecutando...")
}
// Output: ejecutando... → primero → segundo → tercero
💡
Patrón común: defer + mutex.Unlock(), defer + wg.Done(), defer + rows.Close(). Es la forma idiomática de manejar limpieza en Go.

✅ Correcto. Go únicamente tiene for. Se puede usar como bucle clásico, como while, como bucle infinito (for {}), o con range para iterar colecciones.

Funciones

Múltiples retornos, closures, funciones como valores

Las funciones en Go son ciudadanos de primera clase: pueden asignarse a variables, pasarse como argumentos y retornarse como valores. La característica más diferencial es el retorno múltiple de valores.

Declaración básica

funciones.go 
// Función simple
func saludar(nombre string) string {
    return "Hola, " + nombre
}

// Múltiples parámetros del mismo tipo
func sumar(a, b int) int {
    return a + b
}

// Retorno múltiple — signature de Go
func dividir(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("no se puede dividir por cero")
    }
    return a / b, nil
}

// Retornos nombrados
func minMax(arr []int) (min, max int) {
    min, max = arr[0], arr[0]
    for _, v := range arr {
        if v < min { min = v }
        if v > max { max = v }
    }
    return // naked return — retorna min y max
}

// Uso del retorno múltiple
resultado, err := dividir(10, 3)
if err != nil {
    log.Fatal(err)
}
fmt.Printf("%.2f\n", resultado)

// Descartar un valor con _
valor, _ := dividir(10, 2)

Funciones variádicas

variadic.go 
// El último parámetro puede ser variádico con ...
func suma(numeros ...int) int {
    total := 0
    for _, n := range numeros {
        total += n
    }
    return total
}

fmt.Println(suma(1, 2, 3))          // 6
fmt.Println(suma(1, 2, 3, 4, 5))    // 15

// Expandir un slice con ...
nums := []int{10, 20, 30}
fmt.Println(suma(nums...))          // 60

Funciones como valores y closures

closures.go 
// Función anónima asignada a variable
multiplicar := func(a, b int) int {
    return a * b
}
fmt.Println(multiplicar(3, 4)) // 12

// Closure: captura variables del scope exterior
func contador() func() int {
    n := 0
    return func() int {
        n++
        return n
    }
}

c := contador()
fmt.Println(c()) // 1
fmt.Println(c()) // 2
fmt.Println(c()) // 3

c2 := contador() // nuevo contador independiente
fmt.Println(c2()) // 1

// Función de orden superior (middleware pattern)
type HandlerFunc func(string) string

func withLog(fn HandlerFunc) HandlerFunc {
    return func(input string) string {
        fmt.Printf("Input: %s\n", input)
        result := fn(input)
        fmt.Printf("Output: %s\n", result)
        return result
    }
}

procesar := withLog(func(s string) string {
    return strings.ToUpper(s)
})
procesar("hola go") // logs y retorna "HOLA GO"

Funciones init()

init.go 
package main

import "fmt"

var config map[string]string

// init() se ejecuta automáticamente antes de main()

        "host": "localhost",

Arrays, Slices y Maps

Las colecciones más importantes de Go

Arrays

Los arrays en Go tienen tamaño fijo en compilación. Son poco usados directamente; los slices son la herramienta idiomática.

arrays.go 
var nums [5]int                     // [0 0 0 0 0]
ciudades := [3]string{"Bogotá", "Medellín", "Cali"}
matriz := [2][2]int{{1, 2}, {3, 4}} // matriz 2x2

fmt.Println(len(ciudades)) // 3
fmt.Println(ciudades[0])   // "Bogotá"

Slices — La estructura estrella de Go

Un slice es una vista dinámica sobre un array subyacente. Tiene longitud (len) y capacidad (cap). Son la forma idiomática de trabajar con listas en Go.

slices.go 
// Crear slices
s1 := []int{1, 2, 3, 4, 5}
s2 := make([]int, 3)       // [0 0 0] len=3, cap=3
s3 := make([]int, 3, 10)   // [0 0 0] len=3, cap=10

// append — agrega elementos, puede reubicar memoria
s1 = append(s1, 6, 7)
fmt.Println(s1) // [1 2 3 4 5 6 7]

// Slicing — crear sub-slices
a := []int{0, 1, 2, 3, 4, 5}
fmt.Println(a[1:3])  // [1 2]     desde 1 hasta 3 (excluido)
fmt.Println(a[:3])   // [0 1 2]
fmt.Println(a[3:])   // [3 4 5]

// copy — copiar entre slices
dst := make([]int, len(a))
n := copy(dst, a)
fmt.Println(n, dst) // 6 [0 1 2 3 4 5]

// Slice de structs — muy común
type Producto struct {
    ID    int
    Nombre string
    Precio float64
}

productos := []Producto{
    {1, "Café", 5000},
    {2, "Aguardiente", 35000},
}

// Filtrar con slice
caros := productos[:0] // reutiliza memoria
for _, p := range productos {
    if p.Precio > 10000 {
        caros = append(caros, p)
    }
}

// Iterar
for i, p := range productos {
    fmt.Printf("[%d] %s: $%.0f\n", i, p.Nombre, p.Precio)
}

Maps

Un map es una tabla hash que asocia claves únicas con valores. Las claves pueden ser cualquier tipo comparable (string, int, etc.) y los valores cualquier tipo. Su valor zero es nil — debes inicializarlo antes de escribir.

maps.go 
// Crear maps
m1 := map[string]int{"a": 1, "b": 2}
m2 := make(map[string]string)

// Operaciones CRUD
m2["clave"] = "valor"          // Create/Update
val := m2["clave"]             // Read
delete(m2, "clave")            // Delete

// Verificar existencia — ¡importante!
// Sin el segundo valor, obtienes zero value si no existe
nombre, existe := m2["usuario"]
if !existe {
    fmt.Println("No encontrado")
}

// Map de slices — patrón común
agenda := make(map[string][]string)
agenda["lunes"] = append(agenda["lunes"], "standup", "deploy")
agenda["viernes"] = append(agenda["viernes"], "retro")

// Map anidado
config := map[string]map[string]string{
    "db": {"host": "localhost", "port": "5432"},
    "cache": {"host": "redis", "port": "6379"},
}
fmt.Println(config["db"]["host"]) // "localhost"

// Iterar — orden NO garantizado
for k, v := range m1 {
    fmt.Printf("%s: %d\n", k, v)
}
⚠️
Cuidado: Escribir en un map nil produce panic. Siempre inicializa con make() o un literal antes de escribir.

🧠 Pregunta rápida

¿Qué sucede si intentas escribir en un map que es nil en Go?

A) Se crea automáticamente el mapa
B) La escritura se ignora silenciosamente
C) Se produce un panic en runtime
✅ Correcto. Escribir en un map nil causa panic inmediato. Usa make(map[K]V) o un literal map[K]V{} para inicializarlo antes de cualquier escritura.

Structs e Interfaces

OOP a la manera de Go: composición sobre herencia

Go no tiene herencia de clases. En su lugar, usa composición mediante embedding y polimorfismo mediante interfaces implícitas. Este enfoque es más flexible y menos frágil.

Structs y Métodos

structs.go 
package main

import "fmt"

// Definir un struct
type Usuario struct {
    ID     int
    Nombre string
    Email  string
    Activo bool
}

// Constructor (patrón idiomático)
func NuevoUsuario(nombre, email string) *Usuario {
    return &Usuario{
        ID:     generarID(),
        Nombre: nombre,
        Email:  email,
        Activo: true,
    }
}

// Método con receiver de valor — no modifica el original
func (u Usuario) String() string {
    return fmt.Sprintf("Usuario{%d: %s}", u.ID, u.Nombre)
}

// Método con receiver de puntero — puede modificar
func (u *Usuario) Desactivar() {
    u.Activo = false
}

func (u *Usuario) CambiarEmail(email string) error {
    if !strings.Contains(email, "@") {
        return fmt.Errorf("email inválido: %s", email)
    }
    u.Email = email
    return nil
}

// Struct embedding — composición
type Empleado struct {
    Usuario              // embeds Usuario (herencia de comportamiento)
    Cargo   string
    Salario float64
}

func main() {
    u := NuevoUsuario("Elkin", "elkin@hitss.com")
    fmt.Println(u)        // Usuario{1: Elkin}
    u.Desactivar()
    fmt.Println(u.Activo) // false

    e := Empleado{
        Usuario: *u,
        Cargo:   "FullStack Developer",
        Salario: 8_000_000,
    }
    // Los campos y métodos de Usuario son promovidos
    fmt.Println(e.Nombre) // "Elkin" — acceso directo
    fmt.Println(e.String()) // "Usuario{1: Elkin}"
}

Interfaces

En Go las interfaces son implícitas: un tipo implementa una interfaz si tiene todos sus métodos. No se necesita declarar implements.

interfaces.go 
// Definir interfaz
type Almacenable interface {
    Guardar() error
    Buscar(id int) (interface{}, error)

Type Assertions y Type Switch

Una type assertion extrae el valor concreto almacenado dentro de una interfaz. Si el tipo no coincide y no se usa el patrón de dos valores, produce un panic en runtime — por eso siempre se recomienda el patrón comma-ok.

type-assertions.go 
package main

import "fmt"

func main() {
    var i interface{} = "hola go"

    // ── Forma segura: comma-ok pattern ──
    s, ok := i.(string)
    fmt.Println(s, ok)  // "hola go" true

    n, ok := i.(int)
    fmt.Println(n, ok)  // 0 false  ← no hace panic, devuelve zero value

    // ── Forma directa: PANIC si el tipo no coincide ──
    // s2 := i.(string)  // OK
    // n2 := i.(int)     // ← panic: interface conversion

    // ── Type Switch: manejar múltiples tipos idiomáticamente ──
    fmt.Println(describir(42))              // "entero: 42"
    fmt.Println(describir("Go"))            // "string de 2 chars: \"Go\""
    fmt.Println(describir([]int{1, 2, 3})) // "slice de 3 enteros"
    fmt.Println(describir(nil))             // "nil"

    // ── Caso real: leer valor del context o interface{} ──
    var userID interface{} = 123
    if id, ok := userID.(int); ok {
        fmt.Printf("Usuario autenticado: %d\n", id)
    }
}

// Type switch — v.(type) solo es válido dentro de switch
func describir(v any) string {
    switch t := v.(type) {
    case nil:
        return "nil"
    case int:
        return fmt.Sprintf("entero: %d", t)
    case float64:
        return fmt.Sprintf("decimal: %.2f", t)
    case string:
        return fmt.Sprintf("string de %d chars: %q", len(t), t)
    case bool:
        return fmt.Sprintf("bool: %v", t)
    case []int:
        return fmt.Sprintf("slice de %d enteros", len(t))
    case error:
        return fmt.Sprintf("error: %s", t.Error())
    default:
        return fmt.Sprintf("tipo desconocido: %T", v)
    }
}
⚠️
Siempre usa el patrón comma-ok (v, ok := x.(Tipo)) salvo que estés 100% seguro del tipo. Una assertion directa que falla genera un panic en runtime difícil de depurar.
🏆
Principio Go: "Acepta interfaces, retorna tipos concretos". Define interfaces pequeñas (1-3 métodos). La interfaz más usada en Go es io.Reader, que tiene solo 1 método.

Punteros

Referencia vs valor, cuándo y por qué usarlos

Go tiene punteros pero no tiene aritmética de punteros (a diferencia de C). Son seguros y simples. Entenderlos es clave para saber cuándo un método modifica su receptor o no.

Conceptos básicos

punteros.go 
package main

import "fmt"

func main() {
    x := 42
    p := &x   // p es un puntero a x
               // & = "dirección de"
               // * = "dereferenciar (acceder al valor)"

    fmt.Println(x)   // 42
    fmt.Println(p)   // 0xc000014080 (dirección de memoria)
    fmt.Println(*p)  // 42 (valor en esa dirección)

    *p = 100         // modifica x a través del puntero
    fmt.Println(x)   // 100

    // new() — asigna memoria, retorna puntero
    q := new(int)    // *int apuntando a un 0
    *q = 55
    fmt.Println(*q)  // 55
}

// Paso por valor vs paso por referencia
func incrementarValor(n int) {
    n++  // copia local, NO afecta al original
}

func incrementarRef(n *int) {
    (*n)++  // modifica el original
    // también puede escribirse: *n++
}

func main2() {
    x := 10
    incrementarValor(x)
    fmt.Println(x) // 10 — sin cambio

    incrementarRef(&x)
    fmt.Println(x) // 11 — modificado
}

Valor vs Puntero en receivers

Receiver de Valor
valor
func (u Usuario) Nombre() string {
    return u.Nombre
    // copia de u
    // NO puede modificar u
}
Receiver de Puntero ✅
puntero
func (u *Usuario) Activar() {
    u.Activo = true
    // modifica el original
}
📏
Regla práctica:
  • Usa puntero receiver si el método modifica el struct o si el struct es grande (evita copias).
  • Usa valor receiver para structs pequeños inmutables (ej: Point{X,Y}).
  • Si tienes cualquier método puntero, haz todos los métodos de ese tipo puntero.

Punteros a structs en APIs

Goroutines

Concurrencia ligera: el superpoder de Go

Una goroutine es una función que se ejecuta de forma concurrente con otras goroutines en el mismo proceso. Son extremadamente ligeras (~2KB de stack inicial), puedes tener millones activas simultáneamente. Son la base del modelo de concurrencia de Go.

🧵
Goroutine vs Thread: Un thread del OS consume ~1MB de stack. Una goroutine comienza con ~2KB y crece dinámicamente. El runtime de Go multiplexa N goroutines en M threads del OS (modelo M:N).

Lanzar goroutines

goroutines.go 
package main

import (
    "fmt"
    "sync"
    "time"
)

func tarea(nombre string, duracion time.Duration) {
    fmt.Printf("⏳ %s iniciada\n", nombre)
    time.Sleep(duracion)
    fmt.Printf("✅ %s completada\n", nombre)
}

func main() {
    // Sin goroutines — secuencial
    // tarea("A", 2*time.Second) // espera 2s
    // tarea("B", 1*time.Second) // luego espera 1s → total: 3s

    // Con goroutines — concurrente → total: ~2s
    go tarea("A", 2*time.Second)
    go tarea("B", 1*time.Second)
    go tarea("C", 500*time.Millisecond)

    // ⚠️ Sin esperar, main() termina y mata las goroutines
    // Usar WaitGroup para esperar
    time.Sleep(3 * time.Second) // naive — nunca hagas esto en prod
}

sync.WaitGroup — coordinación correcta

waitgroup.go 
func procesarLote(items []string) {
    var wg sync.WaitGroup

    for _, item := range items {
        wg.Add(1)       // Incrementar contador ANTES de la goroutine
        go func(i string) {
            defer wg.Done()  // Decrementar al finalizar
            procesar(i)
        }(item)          // ← pasar item como argumento para evitar closure bug
    }

    wg.Wait() // Bloquea hasta que todos llamen Done()
    fmt.Println("Lote completado")
}

// Limitar concurrencia con semáforo (patrón común)
func procesarConLimite(items []string, maxConcurrencia int) {
    var wg sync.WaitGroup
    sem := make(chan struct{}, maxConcurrencia) // canal como semáforo

    for _, item := range items {
        wg.Add(1)
        go func(i string) {
            defer wg.Done()
            sem <- struct{}{}        // adquirir semáforo
            defer func() { <-sem }() // liberar semáforo
            procesar(i)
        }(item)
    }
    wg.Wait()
}

sync.Once — inicialización única garantizada

sync.Once garantiza que una función se ejecute exactamente una vez, sin importar cuántas goroutines la invoquen simultáneamente. Es el patrón estándar para inicialización lazy de singletons en entornos concurrentes.

sync-once.go 
package main

import (
    "fmt"
    "sync"
)

// Patrón singleton con sync.Once
type Config struct {
    DSN      string
    LogLevel string
}

var (
    config     *Config
    configOnce sync.Once
)

func ObtenerConfig() *Config {
    configOnce.Do(func() {
        // Se ejecuta UNA sola vez, aunque 100 goroutines llamen a ObtenerConfig()
        fmt.Println("Inicializando configuración...")
        config = &Config{
            DSN:      "postgres://localhost/mydb",
            LogLevel: "info",
        }
    })
    return config
}

func main() {
    var wg sync.WaitGroup
    for i := 0; i < 5; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            cfg := ObtenerConfig()
            fmt.Println(cfg.LogLevel)
        }()
    }
    wg.Wait()
    // Output: "Inicializando configuración..." (solo 1 vez)
    //         "info" (5 veces, en cualquier orden)
}

// sync.Once también sirve para cerrar recursos exactamente una vez
var closeOnce sync.Once

func CerrarConexion(conn interface{ Close() error }) {
    closeOnce.Do(func() {
        conn.Close() // se ejecuta exactamente una vez aunque se llame varias veces
    })
}

Channels y Select

"No comuniques compartiendo memoria; comparte memoria comunicando"

Los channels son el mecanismo de comunicación entre goroutines. Son typed pipes: puedes enviar y recibir valores de un tipo específico. El select permite esperar en múltiples canales a la vez.

Channels básicos

channels.go 
// Crear channels
ch := make(chan int)          // Unbuffered — sincrónico
bch := make(chan string, 10)  // Buffered — capacidad 10

// Enviar y recibir
go func() {
    ch <- 42  // enviar (bloquea hasta que alguien reciba)
}()
valor := <-ch // recibir (bloquea hasta que alguien envíe)

// Buffered channel — no bloquea hasta llenarse
bch <- "primero"
bch <- "segundo"
fmt.Println(<-bch) // "primero"

// Cerrar y ranging channels
jobs := make(chan int, 5)
for j := 1; j <= 5; j++ {
    jobs <- j
}
close(jobs) // señalar que no habrá más valores

for j := range jobs {  // itera hasta que se cierre
    fmt.Printf("Procesando job %d\n", j)
}

// Verificar si el channel está cerrado
v, ok := <-jobs
if !ok {
    fmt.Println("Channel cerrado")
}

Pipeline Pattern

pipeline.go 
// Patrón pipeline: encadenar goroutines con channels

func generar(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        for _, n := range nums { out <- n }
        close(out)
    }()
    return out
}

func cuadrado(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        for n := range in { out <- n * n }
        close(out)
    }()
    return out
}

func main() {
    // Encadenar pipeline
    c := generar(2, 3, 4, 5)
    out := cuadrado(c)

    for v := range out {
        fmt.Println(v) // 4, 9, 16, 25
    }
}

context — cancelación y timeouts

El paquete context es fundamental en Go para controlar el ciclo de vida de operaciones: cancelar requests, establecer timeouts y propagar metadatos a través de toda la cadena de llamadas de una API.

context.go 
package main

import (
    "context"
    "fmt"
    "time"
)

func main() {
    // Raíz del árbol de contextos
    ctx := context.Background() // para producción
    // ctx := context.TODO()   // mientras defines el contexto correcto

    // ─── WithCancel — cancelación manual ───
    ctx1, cancel := context.WithCancel(ctx)
    defer cancel() // SIEMPRE con defer — libera recursos aunque no canceles

    go func() {
        time.Sleep(100 * time.Millisecond)
        cancel() // otra goroutine cancela
    }()
    <-ctx1.Done()
    fmt.Println("Cancelado:", ctx1.Err()) // "context canceled"

    // ─── WithTimeout — deadline relativo ───
    ctx2, cancel2 := context.WithTimeout(ctx, 500*time.Millisecond)
    defer cancel2()

    select {
    case <-time.After(1 * time.Second):
        fmt.Println("Operación completada")
    case <-ctx2.Done():
        fmt.Println("Timeout:", ctx2.Err()) // "context deadline exceeded"
    }

    // ─── WithDeadline — fecha/hora absoluta de expiración ───
    deadline := time.Now().Add(2 * time.Second)
    ctx3, cancel3 := context.WithDeadline(ctx, deadline)
    defer cancel3()
    _ = ctx3

    // ─── WithValue — propagar metadatos (userID, requestID, etc.) ───
    // Usa un tipo personalizado como clave para evitar colisiones
    type ctxKey string
    ctx4 := context.WithValue(ctx, ctxKey("userID"), 42)
    ctx4 = context.WithValue(ctx4, ctxKey("requestID"), "abc-123")

    userID := ctx4.Value(ctxKey("userID")).(int)
    fmt.Println("UserID:", userID) // 42
}

// Convención: context siempre es el PRIMER parámetro
func buscarUsuario(ctx context.Context, id int) (*string, error) {
    // Verificar cancelación antes de operaciones costosas
    select {
    case <-ctx.Done():
        return nil, ctx.Err()
    default:
    }
    // Pasar ctx a la DB, HTTP client, etc.
    // db.QueryRowContext(ctx, query, id)
    nombre := "Elkin"
    return &nombre, nil
}

// Handler HTTP — el context de r ya incluye cancelación automática
// Si el cliente desconecta, ctx.Done() se cierra solo
func handler(w http.ResponseWriter, r *http.Request) {
    ctx := r.Context()
    resultado, err := buscarUsuario(ctx, 1)
    if err != nil {
        if errors.Is(err, context.Canceled) {
            return // cliente desconectó, no enviar respuesta
        }
        http.Error(w, err.Error(), 500)
        return
    }
    fmt.Fprintln(w, *resultado)
}
🏆
Reglas de oro del context en Go:
  • Pasa ctx como primer parámetro — nunca lo guardes en un struct
  • Llama siempre cancel() con defer para liberar recursos
  • Propaga el contexto a todas las operaciones de I/O: DB, HTTP, gRPC
  • Usa context.TODO() solo como placeholder temporal

Manejo de Errores

La filosofía Go: errores como valores, no excepciones

Go no tiene excepciones (try/catch). Los errores son valores normales que se retornan y manejan explícitamente. Esto hace que el código sea más predecible y el flujo de errores más visible.

panic y recover

panic-recover.go 
// panic — detiene la ejecución normal (como una excepción no recuperada)
// Solo para errores no recuperables: índice fuera de rango, nil pointer, etc.
// NO usar para control de flujo normal

// recover — recuperar de un panic (solo funciona dentro de defer)
func safeDiv(a, b int) (result int, err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("panic recuperado: %v", r)
        }
    }()
    return a / b, nil // panic si b == 0
}

// Middleware de recuperación para HTTP (¡muy importante en APIs!)
func recoveryMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        defer func() {
            if err := recover(); err != nil {
                log.Printf("PANIC: %v\n%s", err, debug.Stack())
                http.Error(w, "Error interno del servidor", 500)
            }
        }()
        next.ServeHTTP(w, r)
    })
}
⚠️
Regla de oro: Usa error para condiciones esperadas (archivo no encontrado, validación fallida). Usa panic solo para condiciones que indican un bug en el programa (índice fuera de rango, nil pointer inesperado).

    w.WriteHeader(status)
    json.NewEncoder(w).Encode(data)

    responderJSON(w, status, map[string]string{"error": msg})

func obtenerUsuario(w http.ResponseWriter, r *http.Request) {
    id, err := strconv.Atoi(r.PathValue("id")) // Go 1.22+
    if err != nil {
        responderError(w, http.StatusBadRequest, "ID inválido")
        return
    }

    for _, u := range usuarios {
        if u.ID == id {
            responderJSON(w, http.StatusOK, u)
            return
        }
    }
    responderError(w, http.StatusNotFound, fmt.Sprintf("usuario %d no encontrado", id))
}

// POST /usuarios
func crearUsuario(w http.ResponseWriter, r *http.Request) {
    var nuevo Usuario
    if err := json.NewDecoder(r.Body).Decode(&nuevo); err != nil {
        responderError(w, http.StatusBadRequest, "JSON inválido")
        return
    }
    defer r.Body.Close()

    // Validación básica
    if nuevo.Nombre == "" || nuevo.Email == "" {
        responderError(w, http.StatusBadRequest, "nombre y email son requeridos")
        return
    }

    nuevo.ID = len(usuarios) + 1
    usuarios = append(usuarios, nuevo)
    responderJSON(w, http.StatusCreated, nuevo)
}

            return

middleware.go 
// Un middleware es una función que envuelve un Handler
type Middleware func(http.Handler) http.Handler

// Logger middleware
func loggerMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)
        log.Printf("%s %s %v", r.Method, r.URL.Path, time.Since(start))
    })
}

// Auth middleware con Bearer token
func authMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        token := r.Header.Get("Authorization")
        if token != "Bearer mi-token-secreto" {
            http.Error(w, "No autorizado", http.StatusUnauthorized)
            return
        }
        next.ServeHTTP(w, r)
    })
}

// CORS middleware
func corsMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Access-Control-Allow-Origin", "*")
        w.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE, OPTIONS")
        w.Header().Set("Access-Control-Allow-Headers", "Content-Type, Authorization")
        if r.Method == http.MethodOptions {
            w.WriteHeader(http.StatusOK)
            return
        }
        next.ServeHTTP(w, r)
    })
}

// Encadenar middlewares
func chainMiddleware(handler http.Handler, middlewares ...Middleware) http.Handler {
    for i := len(middlewares) - 1; i >= 0; i-- {
        handler = middlewares[i](handler)
    }
    return handler
}

// Uso
mux.Handle("/api/", chainMiddleware(
    apiHandler,
    loggerMiddleware,
    corsMiddleware,
    authMiddleware,

jwt.go 
package middleware

import (
    "net/http"
    "strings"

    "github.com/gin-gonic/gin"
    "github.com/golang-jwt/jwt/v5"
)

var secretKey = []byte("mi-super-secreto-cambiar-en-prod")

type Claims struct {
    UserID int    `json:"user_id"`
    Email  string `json:"email"`
    jwt.RegisteredClaims
}

// Generar token
func GenerarToken(userID int, email string) (string, error) {
    claims := Claims{
        UserID: userID,
        Email:  email,
        RegisteredClaims: jwt.RegisteredClaims{
            ExpiresAt: jwt.NewNumericDate(time.Now().Add(24 * time.Hour)),
            IssuedAt:  jwt.NewNumericDate(time.Now()),
        },
    }
    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    return token.SignedString(secretKey)
}

// Middleware de autenticación
func JWTAuth() gin.HandlerFunc {
    return func(c *gin.Context) {
        header := c.GetHeader("Authorization")
        if !strings.HasPrefix(header, "Bearer ") {
            c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{
                "error": "token requerido",
            })
            return
        }

        tokenStr := strings.TrimPrefix(header, "Bearer ")
        claims := &Claims{}
        token, err := jwt.ParseWithClaims(tokenStr, claims, func(t *jwt.Token) (interface{}, error) {
            return secretKey, nil
        })

        if err != nil || !token.Valid {
            c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{
                "error": "token inválido o expirado",
            })
            return
        }

        // Inyectar claims en el contexto para handlers posteriores
        c.Set("user_id", claims.UserID)
        c.Set("email", claims.Email)
        c.Next()
    }
}

// Usar en handler
func perfil(c *gin.Context) {
    userID := c.GetInt("user_id")
    email := c.GetString("email")
    c.JSON(200, gin.H{"id": userID, "email": email})
}

    if err != nil {
        return nil, err
    }
    db.SetMaxOpenConns(25)
    db.SetMaxIdleConns(25)
    db.SetConnMaxLifetime(5 * time.Minute)

        return nil, fmt.Errorf("ping DB fallido: %w", err)
    }
    return db, nil
}

func (r *UsuarioRepo) ObtenerPorID(ctx context.Context, id int) (*model.Usuario, error) {
    query := `SELECT id, nombre, email, activo, created_at FROM usuarios WHERE id = $1`

    var u model.Usuario
    err := r.db.QueryRowContext(ctx, query, id).Scan(
        &u.ID, &u.Nombre, &u.Email, &u.Activo, &u.CreatedAt,
    )
    if err == sql.ErrNoRows {
        return nil, nil // no encontrado
    }
    if err != nil {
        return nil, fmt.Errorf("obtenerUsuario: %w", err)
    }
    return &u, nil
}

// Transacciones
func (r *UsuarioRepo) TransferirCreditos(ctx context.Context, deOrigen, aDestino, monto int) error {
    tx, err := r.db.BeginTx(ctx, nil)
    if err != nil {
        return err
    }
    defer tx.Rollback() // se ignora si Commit ya se llamó

    if _, err = tx.ExecContext(ctx, `UPDATE cuentas SET saldo = saldo - $1 WHERE id = $2`, monto, deOrigen); err != nil {
        return err
    }
    if _, err = tx.ExecContext(ctx, `UPDATE cuentas SET saldo = saldo + $1 WHERE id = $2`, monto, aDestino); err != nil {
        return err
    }
    return tx.Commit()
}

Generics

Type parameters, constraints y código reutilizable (Go 1.18+)

Los generics introducidos en Go 1.18 permiten escribir funciones y tipos que funcionan con múltiples tipos sin duplicar código. Usan type parameters definidos entre corchetes [T constraint], con constraints que limitan qué tipos se pueden usar.

♻️

Reutilización

Una sola función funciona con int, string, float, structs, etc.

🔒

Type-safe

El compilador verifica tipos en compilación — sin panic ni conversiones.

📜

Constraints

Interfaces que restringen qué tipos puede tomar un type parameter.

⚡

Sin overhead

Go genera código especializado por tipo en compilación.

Funciones genéricas

generics.go 
package main

import "fmt"

// Sin generics — necesitarías una función por cada tipo
func MaxInt(a, b int) int {
    if a > b { return a }
    return b
}

// CON generics — T es el type parameter
// La constraint lista los tipos permitidos
func Max[T int | int64 | float64 | string](a, b T) T {
    if a > b { return a }
    return b
}

// Múltiples type parameters
func Mapa[T, U any](s []T, fn func(T) U) []U {
    resultado := make([]U, len(s))
    for i, v := range s {
        resultado[i] = fn(v)
    }
    return resultado
}

func Filtrar[T any](s []T, fn func(T) bool) []T {
    var resultado []T
    for _, v := range s {
        if fn(v) { resultado = append(resultado, v) }
    }
    return resultado
}

func main() {
    // El compilador infiere el tipo automáticamente
    fmt.Println(Max(3, 5))         // 5    (int)
    fmt.Println(Max(3.14, 2.71))   // 3.14 (float64)
    fmt.Println(Max("b", "a"))     // "b"  (string)

    // Especificar tipo explícitamente (cuando no se puede inferir)
    fmt.Println(Max[int64](10, 20))

    // Funciones de orden superior genéricas
    nums := []int{1, 2, 3, 4, 5}
    dobles := Mapa(nums, func(n int) int { return n * 2 })
    fmt.Println(dobles) // [2 4 6 8 10]

    pares := Filtrar(nums, func(n int) bool { return n%2 == 0 })
    fmt.Println(pares) // [2 4]

    // Cambiar tipo: int → string
    etiquetas := Mapa(nums, func(n int) string {
        return fmt.Sprintf("item-%d", n)
    })
    fmt.Println(etiquetas) // [item-1 item-2 item-3 item-4 item-5]
}

Constraints personalizadas

constraints.go 
// Una constraint es una interfaz con restricción de tipos

// Union de tipos básica
type Numerico interface {
    int | int8 | int16 | int32 | int64 |
    float32 | float64
}

// ~ incluye tipos basados en el tipo subyacente
// ~int acepta: int, y también 'type MiEntero int'
type EnteroLike interface {
    ~int | ~int64
}

func Suma[T Numerico](nums []T) T {
    var total T
    for _, n := range nums { total += n }
    return total
}

// any = interface{} — acepta cualquier tipo
func Primero[T any](s []T) (T, bool) {
    if len(s) == 0 {
        var zero T // zero value del tipo T
        return zero, false
    }
    return s[0], true
}

func main() {
    fmt.Println(Suma([]int{1, 2, 3}))        // 6
    fmt.Println(Suma([]float64{1.1, 2.2}))   // 3.3

    v, ok := Primero([]string{"hola", "go"})
    fmt.Println(v, ok)  // "hola" true

    _, ok = Primero([]int{})
    fmt.Println(ok)     // false
}

Tipos genéricos

generic-stack.go 
// Stack genérico — funciona con cualquier tipo
type Stack[T any] struct {
    items []T
}

func (s *Stack[T]) Push(item T) {
    s.items = append(s.items, item)
}

func (s *Stack[T]) Pop() (T, bool) {
    if len(s.items) == 0 {
        var zero T
        return zero, false
    }
    n := len(s.items) - 1
    item := s.items[n]
    s.items = s.items[:n]
    return item, true
}

func (s *Stack[T]) Len() int     { return len(s.items) }
func (s *Stack[T]) IsEmpty() bool { return len(s.items) == 0 }

func main() {
    // Stack de strings
    var ss Stack[string]
    ss.Push("primero")
    ss.Push("segundo")
    v, _ := ss.Pop()
    fmt.Println(v) // "segundo"

    // Stack de structs
    type Tarea struct { ID int; Titulo string }
    var ts Stack[Tarea]
    ts.Push(Tarea{1, "Escribir tests"})
    ts.Push(Tarea{2, "Deploy a prod"})
    fmt.Println(ts.Len()) // 2
}
⚠️
¿Cuándo usar generics? Úsalos para funciones de utilidad sobre colecciones (Map, Filter, Reduce), tipos contenedor (Stack, Queue, Set) y cuando tienes la misma lógica para múltiples tipos. No los uses para polimorfismo de comportamiento — para eso las interfaces siguen siendo la herramienta correcta.

Reflection y Tooling

Introspección en runtime y herramientas esenciales del ecosistema

El paquete reflect permite inspeccionar y manipular tipos y valores en tiempo de ejecución. Las herramientas del ecosistema Go (go vet, gofmt, linters) garantizan calidad y consistencia en cualquier equipo.

reflect — introspección en runtime

Internamente, encoding/json, database/sql, gin y muchos frameworks usan reflection para leer struct tags y mapear datos dinámicamente. Las dos entidades centrales son reflect.Type (el tipo) y reflect.Value (el valor).

reflect.go 
package main

import (
    "fmt"
    "reflect"
)

type Usuario struct {
    Nombre string `json:"nombre" validate:"required"`
    Email  string `json:"email"  validate:"required,email"`
    Edad   int    `json:"edad"   validate:"min=18"`
}

func main() {
    u := Usuario{Nombre: "Elkin", Email: "elkin@go.dev", Edad: 30}

    // reflect.TypeOf — información del tipo
    t := reflect.TypeOf(u)
    fmt.Println(t.Name())    // "Usuario"
    fmt.Println(t.Kind())    // struct
    fmt.Println(t.NumField()) // 3

    // reflect.ValueOf — el valor
    v := reflect.ValueOf(u)
    fmt.Println(v.Field(0))   // "Elkin"
    fmt.Println(v.Field(1))   // "elkin@go.dev"

    // Iterar campos del struct (lo que hace encoding/json internamente)
    for i := 0; i < t.NumField(); i++ {
        field := t.Field(i)
        value := v.Field(i)
        jsonTag  := field.Tag.Get("json")
        validate := field.Tag.Get("validate")
        fmt.Printf("%-8s json=%-12q validate=%q  valor=%v\n",
            field.Name, jsonTag, validate, value)
    }
    // Nombre   json="nombre"      validate="required"        valor=Elkin
    // Email    json="email"       validate="required,email"  valor=elkin@go.dev
    // Edad     json="edad"        validate="min=18"          valor=30

    // Modificar valores vía reflect (requiere puntero)
    vp := reflect.ValueOf(&u).Elem()
    vp.FieldByName("Nombre").SetString("Elkin González")
    fmt.Println(u.Nombre) // "Elkin González"

    // Verificar si un tipo implementa una interfaz
    errorType := reflect.TypeOf((*error)(nil)).Elem()
    fmt.Println(t.Implements(errorType)) // false
}

Cuándo usar reflection

Usar reflect ✅ Evitar reflect ❌
Librerías genéricas (JSON, validadores, ORMs) Código de negocio común
Procesamiento de struct tags dinámico Cuando generics resuelven el problema
Frameworks de inyección de dependencias Cuando los tipos son conocidos en compilación
Serialización/deserialización dinámica Por conveniencia — impacta performance ~5-10x
⚠️
Reflection tiene costo: Es significativamente más lento que código estático y los errores aparecen en runtime, no en compilación. Úsala solo cuando la flexibilidad dinámica es imprescindible.
